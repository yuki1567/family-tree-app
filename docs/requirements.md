# 家系図アプリ 要件定義書

| 項目 | 内容 |
| --- | --- |
| プロダクト名 | FamilyTree |
| 種別 | Webアプリケーション |
| 主な利用形態 | 個人利用（シングルユーザー / クラウド保存・複数端末からアクセス） |

---

## 1. 背景・目的・ゴール

### 1.1 背景
家族・親族の情報は、紙やメモ、記憶に頼っていると散逸しやすく、世代をまたいで引き継ぐのが難しい。自分の家系を体系的に記録し、ビジュアルに見渡せる手段が求められている。

### 1.2 目的
- 家族・親族の人物情報を詳細に記録・管理する。
- 続柄・婚姻・親子などの関係を、再婚・養子・異母兄弟といった複雑なケースも含めて正確に表現する。
- 家系図としてビジュアルに表示し、関係を直感的に把握できるようにする。

### 1.3 ゴール（成功の定義）
- 複雑な関係（再婚・養子・離婚・異母兄弟など）を、誤りなく記録・表示できる。
- 記録した家系を、家系図としてビジュアルに見渡せる。
- 数千人規模で快適に動作し、数万人規模でも破綻しない。

---

## 2. スコープ

### 2.1 対象範囲（やること）
- ユーザー登録・ログイン認証
- 人物情報の登録・編集・削除
- 人物への写真アップロード
- 家族関係（婚姻・親子）の登録・編集・削除（再婚・養子など複雑なケースを含む）
- 家系図のビジュアル表示と操作
- データのクラウド保存（複数端末からアクセス可能）
- CSVのインポート/エクスポート

### 2.2 対象外（やらないこと）
将来拡張の候補として扱い、初版では対象としない。

- 複数ユーザーでの共同編集・データ共有（ログインはするが、自分のデータを自分だけで扱う。他人と1つの家系図を共同編集する機能は含まない）
- 一般公開・SNS的な機能
- 外部の家系調査サービスとの連携
- モバイルネイティブアプリ（スマホからはWebブラウザでアクセスする）

### 2.3 前提
- データはクラウド（サーバ）に保存する。バックエンドを持つWebサービスとして構築する。
- クラウド保存に伴い、誰のデータかを識別するためにログイン認証を必須とする。
- 利用者は1人だが、その1人がPC・スマホなど複数端末から同じデータにアクセスする形態を指す。
- スマホ対応はWebブラウザ経由とする。
- 写真はサーバ側に保存する。

## 3. 想定ユーザー

| 区分 | 説明 |
| --- | --- |
| プライマリユーザー | 家系図を作成・記録・管理したい個人。自分のアカウントでログインし、PC・スマホなど複数の端末から自分のデータにアクセスする。記録対象は自分の家系に限らず、友人や歴史上の人物などの家系図も作成する。 |

### 3.1 前提
- 利用者は基本的に1人（共同編集・他者との共有はしない）。
- ITリテラシーは一般的なPCユーザーを想定し、家系図やデータ形式の専門知識（GEDCOM等）を持たない前提でUIを設計する。
- 編集・閲覧ともにPC・スマホの両方で行える。スマホでも入力フォームや関係編集が問題なく使えるよう、レスポンシブ対応を本格的に行う。
- 記録する関係は血縁・婚姻の範囲（親子・夫婦・養子・再婚・異母兄弟など）に特化する。師弟・友人といった血縁以外の関係は扱わない。
- ユーザーは「自分の家系図」「友人の家系図」「歴史上の人物の家系図」のように、複数の独立した家系図を作成・切り替えて管理する。

## 4. 機能要件 ― 人物管理

### 4.1 人物に対する操作

| ID | 要件 |
| --- | --- |
| FR-P01 | 人物を新規登録できる |
| FR-P02 | 人物情報を編集できる |
| FR-P03 | 人物を削除できる（削除時、その人物に紐づく関係の扱いを確認・警告する） |
| FR-P04 | 人物の一覧を表示し、氏名で検索・絞り込みできる |
| FR-P05 | 人物に写真を1枚アップロードできる |

### 4.2 人物のプロフィール項目

すべて任意項目とする。

| 項目 | 内容 |
| --- | --- |
| 氏名 | 姓・名を分けず1項目として入力する |
| 性別 | 男性 / 女性 / その他 / 不明 |
| 生年月日 | 正確な日付（年月日）を入力。不明な場合は未入力 |
| 出生地 | 自由記述 |
| 没年月日 | 正確な日付（年月日）を入力。不明な場合は未入力 |
| 写真 | 1枚 |
| 備考 | 自由記述のメモ |

### 4.3 補足
- 全項目を任意とする。家系図では「氏名しか分からない人物」など、情報が部分的にしか得られないケースが普通にあるため。
- 日付はあいまい表現を許可しない。「〜頃」「推定」などのあいまい日付に加え、「年だけ判明／年月だけ判明」も不可とする。日付は正確な年月日を入力するか、分からなければ未入力とする。
- 氏名は姓・名に分けず、1つの項目として扱う。
- 写真は1人につき1枚。
- 旧姓・別名、死没地、存命/故人フラグ、職業、長文プロフィールは対象外とする。短いメモは「備考」で代替する。

## 5. 機能要件 ― 関係管理

複雑な家族関係（再婚・養子・異母兄弟など）を正しく扱うことを目的とする。関係は「夫婦・パートナー関係」と「親子関係」の2種類で構成する。

### 5.1 夫婦・パートナー関係

| ID | 要件 |
| --- | --- |
| FR-R01 | 2人の人物を夫婦・パートナーとして結びつけられる |
| FR-R02 | 1人の人物が複数の相手と関係を持てる（再婚に対応） |
| FR-R03 | 離婚・死別などの関係解消の状態を記録できる |

### 5.2 親子関係

| ID | 要件 |
| --- | --- |
| FR-R04 | 夫婦・パートナー関係の下に、子として人物を登録できる |
| FR-R05 | 親子の種別を「実子 / 養子」から選べる |
| FR-R06 | 片方の親が不明な場合、氏名空欄の「不明な人物」を立ててペアを成立させ、子を登録できる |
| FR-R07 | 異母・異父兄弟を表現できる（同じ親でも相手が違えば別の夫婦・パートナー関係となり、自然に区別される） |

### 5.3 補足
- 夫婦・パートナー関係は異性間のみとする。同性どうしの関係は登録しない。
- 婚外子は扱わない。子は必ず夫婦・パートナー関係の下に登録する。結婚（夫婦関係）を作らずに親子だけを登録することはできない。
- 親子の種別は「実子 / 養子」の2種類とする（継子・里子などは設けない）。
- 関係に日付は持たせない（結婚日・離婚日などは記録しない）。ただし「離婚」「死別」といった関係解消の状態は記録する。
- 片方の親が不明な子は、「不明な人物」（氏名空欄）を1人立てて夫婦・パートナー関係を成立させ、その下に登録する。
- 家系図の最上位世代の人物は、上の世代（親）を登録しなければ「親なし」で自然に成立する。最上位までさかのぼって不明な親を立て続ける必要はない。

---

## 6. 機能要件 ― 家系図表示

記録した家系を家系図としてビジュアルに表示する。数千人規模で快適、数万人規模でも破綻しないという前提（第1章）を踏まえ、一度に全体を描画せず表示範囲を絞る方式を採る。

### 6.1 表示・操作

| ID | 要件 |
| --- | --- |
| FR-V01 | 家系図をツリー状にビジュアル表示する |
| FR-V02 | 中心人物を起点に、上下の一定世代数までを表示する |
| FR-V03 | 表示の端の人物の方向へ移動すると、その先の世代が順次読み込まれ、表示範囲が広がる（中心人物は固定。視点の切り替えは行わない） |
| FR-V04 | ズーム（拡大縮小）・パン（移動）ができる |
| FR-V05 | ノード（人物）をクリックすると、モーダルで詳細の表示・編集ができる |
| FR-V06 | 複数の家系図を切り替えて表示できる |

### 6.2 複数家系図の管理

| ID | 要件 |
| --- | --- |
| FR-V07 | 家系図を複数作成できる |
| FR-V08 | 家系図を切り替えて表示できる |
| FR-V09 | 家系図を編集・削除できる（名称の変更など） |

### 6.3 補足
- 表示は「中心人物から上下一定世代まで」を基本とし、全員を一度に描画しない。端へ移動するたびに先の世代を順次読み込んで表示範囲を広げる。
- 中心人物は固定とし、移動先を新しい中心に置き直す（視点切り替え）はしない。
- 人物の詳細表示・編集は、家系図画面上でノードをクリックして開くモーダルで行う。

## 7. 機能要件 ― インポート/エクスポート

データの受け渡し・一括登録・一括編集のため、CSV形式でのインポート/エクスポートを提供する。表計算ソフトで開ける汎用性を重視する。

### 7.1 エクスポート

| ID | 要件 |
| --- | --- |
| FR-IE01 | 家系図のデータを、家系図1つごとに単一CSVファイルでエクスポートできる |
| FR-IE02 | エクスポートしたCSVは、複雑な関係（再婚・養子・異母兄弟など）を保持できる |
| FR-IE03 | エクスポートしたCSVは、表計算ソフトで開いて一覧として読める |
| FR-IE04 | 写真はCSVに含めない（写真以外のデータのみを出力する） |

### 7.2 インポート

インポートはマージ方式とする。インポートしたCSVの内容を、対象の家系図の既存データに統合する（全置き換えではない）。

| ID | 要件 |
| --- | --- |
| FR-IE05 | CSVをインポートし、対象の家系図の既存データにマージできる |
| FR-IE06 | 同一人物の判定は一意のIDで行う。IDが一致する人物は更新し、IDがない（新規の）行は追加する |
| FR-IE07 | 人物だけでなく、関係（夫婦・パートナー、親子）もマージ対象とする |
| FR-IE08 | CSVに含まれず、アプリ側にのみ存在する既存人物は、削除せずそのまま残す |
| FR-IE09 | インポート時にエラー・警告（書式不正、参照先不明など）を提示する |

### 7.3 補足
- 保存・受け渡し形式はCSVとする（表計算ソフトで開ける汎用性を重視）。
- クラウド保存（第2章）によりバックアップはサーバ側で担保されるため、CSVの主目的はバックアップではなく、一括登録・一括編集・手元での閲覧である。想定する使い方は、初回にまとめてデータを登録する、または既存データをエクスポートして表計算ソフトで一括編集して戻す、など。
- エクスポート/インポートは家系図1つごとに行う（複数家系図を1ファイルにまとめることはしない）。
- 同一人物の判定はID方式とする。氏名等の内容による判定は、同名・改名・氏名空欄の「不明な人物」を区別できないため採用しない。

## 8. 非機能要件

| ID | 分類 | 要件 |
| --- | --- | --- |
| NFR-01 | パフォーマンス | 数千人規模の家系図で、表示・操作が快適に動作する。数万人規模でも破綻しない（一度に全体を描画せず、中心人物から表示範囲を絞る方式で対応する） |
| NFR-02 | 対応ブラウザ | Chrome および Safari の最新版に対応する（Edge / Firefox は対象外） |
| NFR-03 | レスポンシブ | PC・スマホの両方で、閲覧・編集が問題なく行える |
| NFR-04 | セキュリティ | ログイン認証により、本人以外が他人のデータにアクセスできないようにする。通信は暗号化（HTTPS）する。認証方式は設計に委ねる |
| NFR-05 | プライバシー | ユーザーのデータは本人のみがアクセスでき、外部に公開・共有されない |
| NFR-06 | 可用性 | ベストエフォートとする。個人利用のため、厳密な可用性・バックアップ保証は求めない |
| NFR-07 | 国際化 | 初版の表示は日本語のみとするが、将来の多言語対応に備え、画面文言を外部化する（国際化＝i18nの土台を設ける） |

### 8.1 補足
- 対応ブラウザは Chrome / Safari に限定する（Edge・Firefox は対象外）。
- 認証方式（メール＋パスワード、ソーシャルログイン等）は要件として固定せず、設計に委ねる。要件として定めるのは「本人以外がアクセスできないこと」のみ。
- 可用性・バックアップはベストエフォートとし、サーバ側での厳密なデータ保全保証は初版の要件としない。
- 多言語対応そのものは将来拡張だが、後から言語を追加しやすい作り（文言の外部化）を初版の作りに含める。アクセシビリティは標準的な配慮の範囲とする。
